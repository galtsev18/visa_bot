# ADR-0002: TypeScript и JavaScript в одном репозитории

**Status:** Accepted  
**Date:** 2025-03  
**Deciders:** Команда проекта

## Context

Проект изначально на JavaScript. Полный переход на TypeScript за один шаг рискован и трудоёмок. Нужна типизация для портов, use cases и адаптеров, при этом сохранить работоспособность существующего кода в `src/lib/` и `src/commands/` без массового переписывания.

## Decision

Допускаем **сосуществование TypeScript и JavaScript** в `src/`: порты, application и адаптеры — на TypeScript; CLI, lib (домен, оркестратор, HTTP, Sheets, Telegram) — пока на JavaScript. Сборка: `tsc` компилирует только `.ts` в `dist/`; при запуске из `src` используется `tsx` для выполнения `.ts` на лету. Исходники в `src` не импортируют из `dist`. Для use cases из JS (userBotManager) используются реэкспорты через `.js`, указывающие на скомпилированные модули при `npm start` или на исходные `.ts` при `npm run dev` (tsx резолвит).

## Consequences

### Положительные

- Постепенная миграция без «big bang».
- Строгие контракты там, где они важнее всего (порты, сценарии).
- Единый линтер и typecheck для TS-части.

### Отрицательные / риски

- Два языка в одном репо; при импорте TS из JS типы не проверяются на границе (JSDoc может частично помочь).
- Запуск VFS-адаптеров из `src` может требовать tsx или сборки (зависит от резолва динамических импортов).

### Нейтральные

- В [REQUIREMENTS.md](../REQUIREMENTS.md) зафиксировано требование развивать типизацию; полный переход на TS может быть темой следующих ADR.
